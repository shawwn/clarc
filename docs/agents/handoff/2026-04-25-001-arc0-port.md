# Handoff: arc0 CL port ‚Äî 2026-04-25

## What this project is

A port of Paul Graham's Arc language runtime from Racket/Scheme (`arc3.2/ac.scm`) to Common Lisp (SBCL). The motivation is multi-core performance: SBCL has native OS threads (`sb-thread`) while Racket has limited parallelism.

Architecture mirrors dang's description of HN's "Clarc":
- **arc0** (`arc0.lisp`) ‚Äî written in CL; the runtime/compiler (replaces `ac.scm`)
- **arc1** (`arc.arc`) ‚Äî the Arc standard library, loaded on top of arc0
- **arc2** (`srv.arc`, `app.arc`, `html.arc`, `strings.arc`, etc.) ‚Äî application layer

Source arc files live in `../arc3.2/` (sibling directory). The `klarc` shell script boots the system:
```sh
./klarc            # uses ../arc3.2/ by default
ARC_DIR=/path/to/arc3.2/ ./klarc   # override
```

## Current state

**Working:**
- Full custom Arc reader (`arc-read-1`, `arc-read-list`) ‚Äî handles `:` as ssyntax (not CL package separator), `[...]` ‚Üí `(fn (_) ...)`, dotted pairs, char literals including `#\ ` (delimiter chars)
- Arc-to-CL compiler: `ac`, `ac-fn`, `ac-if`, `ac-set`, `ac-call`, `ac-qq` (explicit cons/list/append, no CL quasiquote)
- ssyntax: `:` compose, `.`/`!` sexpr, `~` complement, `&` and
- Global variable table (`*arc-globals*` hash, lowercase string keys)
- Tagged types (CLOS struct `arc-tagged`)
- All Arc primitives: arithmetic, comparison, I/O, tables, strings, coerce, type system
- Networking: `sb-bsd-sockets` ‚Äî `open-socket`, `socket-accept` returns `(in-stream out-stream ip-string)`
- Threading: `sb-thread` ‚Äî `new-thread`, `kill-thread`, `break-thread`, `atomic-invoke`
- Gray stream (`arc-limited-stream`) caps request reads at 2MB
- `arc-boot` loads arc.arc then libs.arc (strings/pprint/code/html/srv/app/prompt)
- HTTP server verified: `HTTP/1.1 200 OK` with correct headers

## Key design decisions / gotchas

- **CL nil = Arc nil**: perfect alignment; no nil-tree conversion needed
- **CL `:` package conflict**: solved by a fully custom reader; never use CL's reader for Arc source
- **SBCL quasiquote**: SBCL uses `SB-IMPL::COMMA` structs internally; `ac-qq` generates explicit `cons`/`list`/`append` CL code instead of CL quasiquote forms
- **`*default-pathname-defaults*`** is bound to arc-dir during boot so relative filenames in `(map load '("strings.arc" ...))` resolve correctly
- **`chars->value`** uses `arc-intern-token` (not CL `read-from-string` with a readtable) for ssyntax token parsing
- **`arc/logs/`** must exist relative to CWD when the server runs ‚Äî `ensure-srvdirs` (called by `serve`, not `serve1`) creates it; the `arc/` directory is gitignored
