# (me) and (ip) helper functions — 2026-04-30

Follow-up cleanup after the user-arg cascade. The cascade itself
([handoff `006`](./2026-04-30-006-user-arg-cascade-complete.md))
left ~30 sites in news.arc / app.arc / blog.arc / prompt.arc still
reading the per-request user via the canonical macro form
`(the me)`, plus ~5 sites repeating `(is (the me) X)` to test
"am I `X`?". This handoff introduces two small helper functions
that read more naturally than the macro form, and sweeps both
patterns across the codebase.

## What was the goal

Make calls into the per-request thread-locals read as plain
function calls — `(me)` instead of `(the me)`, `(ip)` instead of
`(the ip)` — and collapse the most common comparison idiom
`(is (the me) X)` into a single 1-arg `(me X)` that returns the
username when matching and nil when not.

The 1-arg form is the more interesting piece. It's a value-AND-
predicate: usable in `(when (me user) ...)` as a boolean check,
and inside `(awhen (me user) ...)` as a username-binding. Same
function call covers both shapes.

## The (me) function — 0-arg and 1-arg

In [`app.arc`](../../../app.arc):

```arc
(def me ((o other))
  (let m (the me)
    (if other (and (is m other) m) m)))
```

Two modes:

- **`(me)`** — returns `(the me)`. A direct shorthand. Saves 5
  characters and reads as a function call rather than a macro.
- **`(me other)`** — returns `(the me)` if it equals `other`,
  else nil. Replaces the recurring idiom `(is (the me) X)` /
  `(is X (the me))`.

The 1-arg form deliberately returns the username (not a boolean):
it works as both a predicate and a value-extractor:

```arc
;; before
(when (is (the me) user)
  (do-thing user))

;; after
(awhen (me user)
  (do-thing it))
```

`(me other)` returns the username on match, nil on mismatch. The
`when` / `awhen` / `aif` family all treat any non-nil as truthy,
so the dual use is natural.

### Where (me other) replaced (is (the me) X)

- `saved-link`: `(when (or (admin) (me user)) ...)` — show saved
  links for self or admin
- `savedpage`: `(if (or (me user) (admin)) ...)` — same
- `threads-page`: `(if (me user) "threads" title)` — label is
  "threads" when viewing your own page, the username's "comments"
  otherwise
- `user-fields`: `w (me user)` flag for "is the field-target the
  viewer" — used to gate which fields are editable
- `prompt.arc edit-app`: `(if (me me) ...)` — yes, this reads
  oddly because the captured local is also named `me`. The
  shadowing is intentional and works because `(t me)` binds a
  local before the function body runs.

The `when-umatch` / `when-umatch/r` macros also expand to
`(me ,user)` instead of `(is ,user (the me))`.

## The (ip) function — 0-arg only

In [`app.arc`](../../../app.arc):

```arc
(def ip () (the ip))
```

Single-arg form intentionally not provided. The pattern
`(is (the ip) X)` doesn't repeat in the current code, and an IP
match isn't typically a useful predicate (you'd use a session-
level identity check via `(me X)` instead). If a future use case
demands it, the symmetric 1-arg form is a small addition.

### Where (ip) replaced (the ip)

- `newslog` body
- `oversubmitting` recent-items check
- `comment-ban-test` banned-ips check
- `note-baditem` (3 sites)
- `create-story` / `create-poll` / `create-pollopt` /
  `create-comment` `'by ... 'ip ...` paired item-record fields
  (kept the pairing — both halves use the helper form for
  symmetry)
- `app.arc`'s `login-handler`, `create-handler`, `whoami`,
  `hello-page`
- `news.arc`'s `opexpand` binding `(with (user (me) ip (ip)) ...)`

## w/me moved from arc.arc to app.arc

Previously [`arc.arc`](../../../arc.arc) defined `w/me` as a
shorthand for `(w/the me val ...)`, sitting next to the
`(the var)` / `(t var)` / `w/the` primitives. After this
commit, `w/me` lives in [`app.arc`](../../../app.arc) next to
`(me)` and `(ip)` — i.e., everything that's specific to "the
request has a user" rather than the underlying generic
thread-local mechanism.

This also simplifies arc.arc: it now only contains the language-
level primitive (`thread-local`, `the`, `w/the`, `(t var)`),
and the request-specific affordances (`me`, `ip`, `w/me`) are in
the application-server layer where they belong.

The cost is that `test.arc`'s test for `w/me` had to convert.
The test is now `thread-local-w/the-me` and exercises `(w/the me
...)` directly. This is arguably more correct: the test was
testing the underlying primitive, and `w/me` was just a
not-very-interesting wrapper around it. The test now exercises
the actual mechanism.

## Sharp edge — CSRF capture rule for (me)

This is **the** thing future agents need to know about `(me)`.

`(me)` is a function. Calling it inside a closure body re-reads
the thread-local *at closure execution time*, not at the time
the closure was constructed. For fnid closures (forms, links),
this means:

- **At render time**, `(the me)` is the renderer (the user being
  shown the form/link).
- **At submit time**, `(the me)` is whoever clicked / submitted.
  In practice usually the same person, but not always (consider
  a shared private link).

The CSRF check in `when-umatch` and similar relies on capturing
the **render-time** user into a lexical variable that the closure
body closes over. Compare:

```arc
;; CORRECT — captures render-time user into closure
(let user (me)
  (aform (when-umatch user
           (do-action))
    body))

;; WRONG — closure body re-reads at submit time;
;; (when-umatch (me) ...) trivially succeeds because it's
;; comparing the submitter to the submitter
(aform (when-umatch (me)
         (do-action))
  body)
```

The macros [`uform`](../../../app.arc) and [`urform`](../../../app.arc)
encapsulate this pattern internally — both expand to
`(let ,g (me) ...)` and the fnid thunk closes over `g`.

Manual call sites that build their own fnid closures (`admin-page`,
`vars-form`, `prompt-page`) follow the same pattern explicitly:

```arc
(def admin-page msg
  (let user (me)        ; capture once at render time
    (whitepage
      ...
      (aform (when-umatch user ...)
        ...))))
```

If you find yourself adding a fnid closure that needs to know
"who rendered this", do `(let user (me) ...)` first. Don't call
`(me)` inside the closure body for that purpose.

This is the same gotcha documented in [handoff
`004`](./2026-04-30-004-dang-user-subject-cascade.md) and
[handoff `006`](./2026-04-30-006-user-arg-cascade-complete.md)
for `uform` / `urform`, restated here because the bare function
form `(me)` looks innocuous compared to the macro form `(the me)`
and is easier to misuse.

## What stayed as (the me) / (the ip)

Not everything got converted, intentionally:

- **Setters** `(= (the me) user)` / `(= (the ip) ...)` — `(me)`
  and `(ip)` are functions, not places. The setforms only attach
  to the underlying `thread-local` accessor through the `(the var)`
  macro.
- **Inside `(def me ...)` and `(def ip ...)` themselves** —
  using `(me)` would infinite-recurse.
- **`test.arc` thread-local primitive tests** — the tests
  exercise the underlying machinery (`thread-local`, `the`,
  `w/the`). They should test what they claim to test.
- **Doc comments** documenting the primitive (`arc.arc`,
  `srv.arc`, `app.arc`'s `vars-form` capture comment, news.arc's
  ignore-actor comment) — describing the underlying mechanism is
  more useful than describing the helper.

After this sweep the only `(the me)` / `(the ip)` references in
the codebase are setters, recursion-prevention inside the helper
defs themselves, primitive tests, and prose comments.

## Where to look for naming concerns

`me` is now a **global function** in app.arc. It does not collide
with the existing uses of `me` as a name because:

- `me` as a *thread-local key* in `(the me)` / `(t me)` /
  `(w/the me ...)` — those are macro forms that quote the symbol
  `me` literally; they don't reference the function.
- `me` as a *local variable name* in `(let me ...)`, `(t me ...)`
  parameter form, `(fn (me) ...)` — locals shadow the global. The
  function `me` is only in scope when no local binding shadows it.
  Inside `(t me me)`-style parameter forms (e.g.
  prompt.arc:`edit-app`), the local binding wins.

If you're debugging an "expected the function but got the local"
or vice-versa: check the closest enclosing binding form. The
shadowing rule is standard arc lexical scope; nothing magic.
