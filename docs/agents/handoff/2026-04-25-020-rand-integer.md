## Handoff 020: `(rand N)` returns integer in [0, N) — 2026-04-25

`(rand)` already returned a float in [0.0, 1.0), but `(rand N)` was
also returning a float — the implementation coerced N to a double
before calling `random`. In Arc the integer-N form should return an
integer in [0, N), matching uses like `(rand (len seq))` in
`rand-elt` (arc.arc:1330) and `(rand (len h))` in `rand-key`
(arc.arc:1647).

## Fix

`arc0.lisp:1189` — drop the `coerce ... 'double-float)` so we just
call `(random n)`. Common Lisp's `random` preserves N's type:
integer N → integer result, float N → float result. The no-arg
branch still uses `1.0d0` to keep the float behavior.

## Verified

```
arc> (rand)
0.6294471125383416d0
arc> (rand 10)
9
arc> (rand 100)
58
```

## Files changed

- `arc0.lisp:1189-1191` — `rand` returns integer when N is an integer.
