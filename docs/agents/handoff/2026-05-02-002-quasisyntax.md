# `#`...#,...` quasisyntax â€” 2026-05-02

## The problem

Arc's `ac` compiler walks every form and recursively compiles
subexpressions: a literal `(m)` becomes a function-call form
`(arc-call0 (arc-global-ref 'arc::|m|))` rather than the data list `(m)`.
That's correct for ordinary arc, but it breaks when the surrounding
operator is a *CL macro* that wants its arguments unevaluated.

Concrete example: `sb-thread::with-mutex` expects `(mutex &key ...)` as
its first form, where `mutex` is a Lisp expression evaluating to the
lock. From arc:

```arc
(sb-thread::with-mutex (m) body)
```

`ac` compiles the inner `(m)` as a function call, the macro receives
something like `((arc-call0 (arc-global-ref ...)))` instead of `(m)`,
and chokes when it tries to destructure a function-call form as a
binding spec.

## The fix

Reader-side syntax was already there (`#`` and `#,` produce
`quasisyntax` / `unsyntax` symbols at [`arc1.lisp:266,273`](../../../arc1.lisp));
this commit adds the compiler clauses.

`#`form` â‡’ `(quasisyntax form)` â‡’ `ac` returns `form` *as a literal CL
expression*, walking it but leaving every subform alone â€” except for
`#,expr` â‡’ `(unsyntax expr)` holes, which are replaced by `(ac expr env)`.

```arc
(let v 41
  #`(cl::let ((y #,v)) y))
;; ac produces literally:
;; (cl::let ((y arc::|v|)) y)
;; CL macroexpands `let` normally; runtime value is 41.
```

## Implementation

[`arc1.lisp ac`](../../../arc1.lisp) â€” three new dispatch clauses next
to the existing `quote` / `quasiquote`:

```lisp
((arc-sym= (arc-car? s) "quasisyntax") (ac-qs (cadr s) env))
((arc-sym= (arc-car? s) "unsyntax")
 (error "unsyntax outside quasisyntax: ~S" s))
((arc-sym= (arc-car? s) "unsyntax-splicing")
 (error "unsyntax-splicing outside quasisyntax: ~S" s))
```

The walker:

```lisp
(defun ac-qs (x env)
  (cond
    ((not (consp x)) x)
    ((arc-sym= (car x) "unsyntax") (ac (cadr x) env))
    ((arc-sym= (car x) "unsyntax-splicing")
     (error "unsyntax-splicing inside quasisyntax not supported: ~S" x))
    (t (cons (ac-qs (car x) env) (ac-qs (cdr x) env)))))
```

That's the whole compiler change. No nesting / level-counting; if it
ever bites the level-counter pattern from `ac-qq1` ports over directly.

## Why no `unsyntax-splicing`

`#,@expr` would force the result into a runtime list-construction
expression (some `(append ...)` form). That defeats the entire purpose:
the *compile-time literal form* is what lets the surrounding CL macro
see the source it expects. If we built the form at runtime, CL would
evaluate the list-builder expression, not macroexpand a known shape.

The workaround is small enough that not supporting splicing isn't a
hardship: wrap the body in a `do` and unsyntax it once.

```arc
;; Don't:
#`(cl::with-mutex (#,m) #,@body)

;; Do:
#`(cl::with-mutex (#,m) #,(do ,@body))
```

`do` in arc compiles to `progn` in CL, which sequences the body forms
identically. The CL macro sees a single form in the body slot â€” same
shape, same effect.

## What now works

CL special forms and macros that take literal forms:

```arc
(test? 7 #`(cl::let ((y 7)) y))
(let v 41
  (test? 41 #`(cl::let ((y #,v)) y)))
(test? "hi 42" #`(cl::with-output-to-string (s)
                   (cl::princ "hi " s)
                   (cl::princ #,(+ 40 2) s)))
```

The `with-output-to-string` test is the one that earns its keep:
`s` is a stream binding the CL macro introduces, and it needs to stay
literal across the inner `princ` calls. `#,(+ 40 2)` punches a hole
that ac compiles (the arc expression evaluates at runtime to 42),
while `s` outside any unsyntax stays as the literal symbol both the
binding form and the consumers see.

## Sequel: this unblocks the rwlock wrapper

The motivating use case was a reader-writer-lock wrapper over
`sb-thread::with-mutex` + two semaphores. The previous
[handoff](./2026-05-02-001-setf-arc-global-ref.md) made arc globals
setf-able from CL; this one makes CL macros callable from arc. With
both in place the rwlock can be written cleanly as:

```arc
(mac w/read-lock (rw . body)
  (w/uniq g
    `(let ,g ,rw
       (sb-thread::wait-on-semaphore (,g 'turnstile))
       (sb-thread::signal-semaphore  (,g 'turnstile))
       #`(sb-thread::with-mutex (#,(,g 'm))
           #,(do (when (is (car (,g 'readers)) 0)
                   (sb-thread::wait-on-semaphore (,g 'ws)))
                 (++ (car (,g 'readers)))))
       (after (do ,@body)
         #`(sb-thread::with-mutex (#,(,g 'm))
             #,(do (-- (car (,g 'readers)))
                   (when (is (car (,g 'readers)) 0)
                     (sb-thread::signal-semaphore (,g 'ws)))))))))
```
