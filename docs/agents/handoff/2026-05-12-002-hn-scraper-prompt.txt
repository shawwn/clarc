Come up with a way of loading items and users from the real Hacker News. Ideally it would load the first two pages of HN somehow. (So it would fetch the current front page and second-page, then for each item, import the item, which imports all comments, which imports all users. If users already exist, they're updated to match the current HN user.)

Note that the HN API doesn't give information about whether a comment
is marked dead, or whether it's collapsed, or flagged, or killed. I
think we'd want to come up with a way to scrape the front page
ourselves, and export each item and user to a json file. It should
respect the crawl-delay in https://news.ycombinator.com/robots.txt.
It'd be a nice exercise to write it in Arc. Feel free to extend Arc as
needed.
